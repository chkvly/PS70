System Overview

Hardware Components

• Microcontroller: An ESP32 microcontroller will be used for its WiFi capabilities, enabling it to connect to the internet and communicate with a serverless database. It will also manage the seat detection sensors.

• Seat Detection Sensors: A combination of resistor sensing and capacitive sensors will be used to detect the presence or absence of a person in a seat. The choice between the two will be made based on a comparison of their sensitivity, power consumption, and ease of integration.

• Indicator LED: Each seat will have an LED indicator to provide a visual cue of its status (e.g., green for available, red for occupied).

Software Components

• Serverless Database (Firebase): Google's Firebase will serve as the backend, storing real-time data on seat status (available, filled, pending). This choice is motivated by Firebase's scalability, ease of setup, and real-time data synchronization features.

• Front-End Webpage: A dedicated webpage will serve as the user interface, displaying available seats in real-time. The page will be designed with responsiveness in mind, ensuring accessibility across various devices and screen sizes.

• QR Code Generation: To facilitate user interaction, each seat will be associated with a QR code. Upon leaving a seat, users can scan the QR code, which prompts them to confirm if they wish to make the seat available. This feature will be implemented using a QR code generating API.

Implementation Plan

1. Hardware Setup: Assemble the microcontroller and sensors for a prototype seat. Develop the circuitry for seat detection and the LED indicator.

2. Software Development:

• Set up the Firebase database, defining the structure to store seat statuses.

• Develop the code for the ESP32 to read sensor data and communicate with Firebase, updating the seat status accordingly.

• Design and implement the front-end webpage, ensuring it dynamically displays seat availability based on the database.

• Integrate QR code generation, linking each seat to a unique code that users can interact with.

Testing and Validation

• Prototype Testing: Test the functionality of the prototype seat with the microcontroller and sensors, ensuring accurate detection of seat occupancy.

• System Integration Test: Validate the seamless operation of the entire system, from hardware sensing and data transmission to real-time display on the webpage.

• User Feedback: Conduct a pilot run in a controlled environment, gathering user feedback on the system's usability and effectiveness.
